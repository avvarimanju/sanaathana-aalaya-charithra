🧪 STEP 4: TEST API ENDPOINTS DIRECTLY (OPTIONAL)
═══════════════════════════════════════════════════════════════════════════════

You can test the API directly using PowerShell:

# List all sources
Invoke-RestMethod -Uri "http://localhost:4000/api/admin/trusted-sources" -Method GET

# Get single source
Invoke-RestMethod -Uri "http://localhost:4000/api/admin/trusted-sources/source_001" -Method GET

# Create new source
$body = @{
    sourceName = "Test Temple Website"
    sourceUrl = "https://example.com"
    sourceType = "temple_official"
    trustScore = 8
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:4000/api/admin/trusted-sources" -Method POST -Body $body -ContentType "application/json"

Problem: "Failed to fetch trusted sources"
Solution: Make sure local backend is running on port 4000

Problem: "No sources displayed"
Solution: Check browser console for errors, verify API URL in .env.development

Problem: Backend won't start
Solution: Run "npm install" in project root first
