Request Flow Diagram

1. User Login Flow:
  Frontend → Cognito User Pool → MFA Verification → JWT Token
  → Session Created in AdminSessions table

2. API Request Flow:
  Frontend (with JWT) → API Gateway → Custom Authorizer
  → Rate Limiter Check → Session Validation
  → Admin API Lambda → Handler (Temple/Artifact/etc.)
  → DynamoDB/S3 → Response → Frontend

3. Temple Creation Flow:
  Frontend Form → POST /admin/temples → Admin API Lambda
  → Temple Handler → Validation → DynamoDB (HeritageSites)
  → Audit Log Entry → Success Response

4. Image Upload Flow:
  Frontend → Base64 Encode → POST /admin/temples/{id}/images
  → Temple Handler → S3 Upload → URL Generation
  → Update Temple Record → Return S3 URL

5. Bulk Operations Flow:
  Frontend → POST /admin/temples/bulk-delete
  → Temple Handler → Iterate siteIds
  → Soft Delete Each → Track Success/Failures
  → Return Results Summary

6. Audit Trail Flow:
  Any Write Operation → Audit Log Helper
  → Capture Before/After Values → DynamoDB (AuditLog)
  → TTL Set (365 days)
