================================================================================
                    CONTENT FORMATS - SIMPLE SUMMARY
================================================================================

YOUR QUESTION:
"Pre-generated content will be in which format? Text, audio, video, 
infographics? How do we choose between them?"

SIMPLE ANSWER:
Only TEXT is pre-generated. Everything else is either on-demand or manual.

================================================================================
                         WHAT GETS PRE-GENERATED
================================================================================

✓ TEXT ONLY
  - Generated by: AWS Bedrock (AI)
  - When: Before users access the app
  - Sections: About, History, Significance, Architecture
  - Cost: $9.20 for 1,000 temples (one-time)
  - Storage: DynamoDB

================================================================================
                    WHAT GETS GENERATED ON-DEMAND
================================================================================

✓ AUDIO (Text-to-Speech)
  - Generated by: Google Cloud TTS
  - When: User clicks "Play Audio" button
  - Languages: 9 Indian languages
  - Cost: $0.000016 first time, FREE after caching
  - Storage: S3 (cached)

================================================================================
                    WHAT YOU UPLOAD MANUALLY
================================================================================

✓ PHOTOS
  - Upload via: Admin Portal
  - When: Anytime (before or after launch)
  - Cost: Storage only (~$1.10/month for 1,000 temples)

✓ VIDEOS
  - Option A: YouTube links (RECOMMENDED - FREE)
  - Option B: Upload to S3 (EXPENSIVE - $330/month)

✗ INFOGRAPHICS (Optional)
  - Create in: Canva/Figma
  - Upload via: Admin Portal
  - Cost: Storage only (~$0.60/month)

================================================================================
                         HOW TO CHOOSE
================================================================================

YOU DON'T CHOOSE - IT'S AUTOMATIC!

Here's what happens:

1. TEXT: Always pre-generated (required)
   → You run the generation script
   → Bedrock creates text content
   → Saved to database

2. AUDIO: Automatically generated when user requests
   → User clicks "Play Audio"
   → System checks if audio exists
   → If not, generates it
   → Caches for next user

3. PHOTOS: You decide how many to upload
   → Upload 2-3 photos per temple (minimum)
   → Upload 5-10 photos per temple (recommended)
   → Upload 10+ photos per temple (premium)

4. VIDEOS: You decide if you want them
   → Add YouTube links (FREE)
   → OR upload videos to S3 (EXPENSIVE)
   → OR skip videos entirely

5. INFOGRAPHICS: Optional
   → Create if you want visual elements
   → Skip if text and photos are enough

================================================================================
                         COMMON QUESTIONS
================================================================================

Q: Do I need to choose which format to generate?
A: No! Text is always generated. Audio is automatic on-demand. 
   Photos/videos are manual uploads.

Q: Can I generate only text and add audio later?
A: Yes! You can enable audio anytime by setting environment variable.

Q: Do I need to generate audio for all languages?
A: No! Audio is generated only when user requests it in their language.

Q: What if I don't upload photos?
A: App still works, but looks less appealing. Minimum 2-3 photos recommended.

Q: Should I upload videos to S3 or use YouTube?
A: Use YouTube! It's FREE. S3 video storage is very expensive ($330/month).

Q: Can I skip infographics?
A: Yes! They're optional. Text and photos are usually enough.

Q: How do I control costs?
A: 
   - Use Sonnet for text (one-time cost)
   - Enable audio (minimal cost, great UX)
   - Upload 2-3 photos per temple (low cost)
   - Use YouTube for videos (FREE)
   - Skip infographics (optional)
