# Temple ID Format Guide

Standardized format for generating unique temple IDs.

**Last Updated**: March 3, 2026  
**Status**: ID Format Specification

---

## Format Structure

```
STATE-CODE_DISTRICT-CODE_CITY-CODE_TEMPLE-NAME
```

### Components

1. **STATE-CODE**: 2-letter state code (ISO 3166-2:IN)
2. **DISTRICT-CODE**: 3-letter district code
3. **CITY-CODE**: 3-letter city/town code
4. **TEMPLE-NAME**: Kebab-case temple name

---

## Benefits of This Format

1. **Unique**: Each temple has a unique ID
2. **Hierarchical**: Shows state → district → city → temple
3. **Searchable**: Easy to filter by state, district, or city
4. **Sortable**: Temples automatically group by location
5. **Human-readable**: Easy to understand the location
6. **URL-friendly**: Can be used in URLs
7. **Database-friendly**: Good for indexing and querying

## Generation Script

I'll create a script to automatically generate temple IDs:

```powershell
# Generate temple ID
function Generate-TempleId {
    param(
        [string]$State,
        [string]$District,
        [string]$City,
        [string]$TempleName
    )
    
    $stateCode = Get-StateCode $State
    $districtCode = Get-DistrictCode $District
    $cityCode = $City.Substring(0, 3).ToUpper()
    $templeSlug = $TempleName.ToLower() -replace '\s+', '-' -replace '[^a-z0-9-]', ''
    
    return "${stateCode}_${districtCode}_${cityCode}_${templeSlug}"
}
```

---

## Validation Rules

1. **State Code**: Must be valid 2-letter ISO code
2. **District Code**: Must be 3 uppercase letters
3. **City Code**: Must be 3 uppercase letters
4. **Temple Name**: Must be kebab-case (lowercase, hyphens only)
5. **Total Length**: Typically 30-60 characters
6. **Uniqueness**: Must be unique across all temples

## Migration from Old Format

If you have existing temple IDs in old format (`temple_001`), use this migration:

```javascript
// Old format
"templeId": "temple_001"

// New format
"templeId": "TN_THA_THA_brihadeeswarar-temple",
"legacyId": "temple_001"  // Keep for backward compatibility
```
