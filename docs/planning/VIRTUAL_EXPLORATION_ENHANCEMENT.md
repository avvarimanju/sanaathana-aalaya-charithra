# Virtual Exploration Enhancement
## Browse & Explore Temples from Home

---

## 🎯 Enhancement Overview

**Goal:** Allow users to explore temples and artifacts from home without physical visit or QR scanning.

**Use Cases:**
1. 🏠 **Pre-Visit Planning** - Research before visiting
2. 🧓 **Elderly/Disabled** - Can't visit physically
3. 📚 **Students** - Educational research
4. 🌍 **International Users** - Can't travel to India
5. 💾 **Offline Preparation** - Download before visiting remote temples

---

## 🆕 New Features

### **1. Temple Browser**
- Browse all 11 temples
- View temple details, location, timings
- See artifact count and highlights
- Filter by state, type, popularity

### **2. Artifact Explorer**
- Browse all artifacts within a temple
- View artifact details without QR scan
- Access full content (audio, video, text)
- Search across all artifacts

### **3. Virtual Tour Mode**
- 360° temple views (future)
- Artifact gallery
- Guided virtual tours
- Interactive maps

### **4. Download Manager**
- Download temple content for offline
- Manage downloaded content
- Storage usage tracking
- Auto-update downloaded content

### **5. Favorites & Collections**
- Save favorite temples
- Create custom collections
- Share collections with friends
- Bookmark artifacts

## 🔧 Technical Implementation

### **New API Endpoints**

```typescript
// Temple Browsing
GET /api/temples
  - List all temples
  - Query params: state, type, search
  - Response: Temple list with metadata

GET /api/temples/{templeId}
  - Get temple details
  - Response: Full temple info + artifact list

GET /api/temples/{templeId}/artifacts
  - List all artifacts in temple
  - Response: Artifact list with previews

// Artifact Access (Without QR)
GET /api/artifacts/{artifactId}
  - Get artifact details
  - Response: Artifact info + content URLs

POST /api/artifacts/{artifactId}/unlock
  - Unlock artifact content
  - Body: { paymentToken, userId }
  - Response: Full content access

// Bulk Download
POST /api/download/temple/{templeId}
  - Request temple download package
  - Response: Download URLs for all content

GET /api/download/status/{downloadId}
  - Check download status
  - Response: Progress, URLs

// Collections
GET /api/collections
  - Get user's collections
  - Response: Collection list

POST /api/collections
  - Create new collection
  - Body: { name, description, items }

PUT /api/collections/{collectionId}
  - Update collection
  - Body: { name, items }

// Favorites
POST /api/favorites
  - Add to favorites
  - Body: { itemType, itemId }

GET /api/favorites
  - Get user favorites
  - Response: Favorite items
```

## 🔐 Access Control Logic

```typescript
// Check if user has access to content
async function hasAccess(userId: string, artifactId: string): Promise<boolean> {
  // 1. Check if user has active subscription
  const subscription = await getActiveSubscription(userId);
  if (subscription) return true;
  
  // 2. Check if user purchased this temple
  const artifact = await getArtifact(artifactId);
  const templePurchase = await getTemplePurchase(userId, artifact.siteId);
  if (templePurchase && !isExpired(templePurchase)) return true;
  
  // 3. Check if user purchased this specific artifact
  const artifactPurchase = await getArtifactPurchase(userId, artifactId);
  if (artifactPurchase && !isExpired(artifactPurchase)) return true;
  
  return false;
}
```
