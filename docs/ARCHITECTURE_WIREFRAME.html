8 Major Components

1. Mobile App (React Native/Expo)

Location: mobile-app/
User-facing Android application providing the primary interface for temple visitors.
  • QR code scanning interface
  • Audio guide playback
  • Video player with subtitles
  • Infographic viewer
  • Q&A chat interface
  • 10+ language support
  • Offline capability
  • Payment integration (Razorpay)

2. Admin Dashboard (React/TypeScript)

Location: admin-dashboard/
Web-based administrative interface for defect tracking and issue management.
  • View all defect reports
  • Filter by status (New, Acknowledged, In Progress, Resolved, Closed)
  • Search defects by ID or title
  • Update defect status with workflow validation
  • Add status updates and comments
  • Admin authentication (JWT)
  • Audit trail logging

3. Admin Backend (Python Lambda) - NEW

Location: src/admin/
Comprehensive administrative backend APIs for platform management.
Status: 50% Complete (7/14 tasks)
✅ Tasks 1-7: Infrastructure, Auth, Temple/Artifact Management, Content Monitoring, Analytics
🚧 Tasks 8-14: User Management, Configuration, Moderation, Cost Monitoring, Payments, Logging
  • 35+ REST API endpoints
  • Temple CRUD + bulk operations (8 endpoints)
  • Artifact CRUD + QR generation (8 endpoints)
  • Content job monitoring (5 endpoints)

  • JWT authentication + rate limiting
  • Audit logging (365-day retention)

4. Core Backend (TypeScript Lambda)

Location: src/lambdas/
Primary backend handling user-facing operations and content delivery.
  • QR code processing
  • Content generation orchestration
  • Q&A processing with RAG
  • Analytics event tracking
  • Payment handling (Razorpay)
  • Session management

5. Pre-Generation System (TypeScript)

Location: src/pre-generation/
Batch content generation system for cost optimization.
  • Batch processes all 49 artifacts
  • Progress tracking with recursive Lambda
  • Handles Lambda timeout limits
  • Cost optimization (~₹186 one-time vs ongoing)
  • Audio, video, infographic generation
  • Multi-language support

6. Social Authentication (Python Lambda)

Location: src/auth/
OAuth integration for social login providers.
  • Google OAuth integration
  • Facebook OAuth integration
  • Apple Sign-In integration
  • User session management
  • Token refresh handling
  • Secure credential storage (AWS Secrets Manager)

7. Defect Tracking System (TypeScript)

Location: src/defect-tracking/
User issue reporting and resolution workflow management.
  • User defect submission from mobile app
  • Status workflow: New → Acknowledged → In Progress → Resolved → Closed
  • Admin status updates with comments
  • Status transition validation
  • Defect search and filtering
  • Status history tracking
  • Admin authentication required for updates

8. Infrastructure (AWS CDK)

Location: infrastructure/
Infrastructure as Code managing all AWS resources.
  • DynamoDB table definitions
  • Lambda function configurations
  • API Gateway setup
  • S3 bucket + CloudFront CDN
  • IAM roles and policies
  • CloudWatch logging
  • Deployment automation

Data Flow Diagrams

User Content Request Flow

1. User Scans QR Code
Mobile app captures QR code data at temple artifact
↓
2. API Gateway
POST /qr → Routes to QR Processing Lambda
↓
3. QR Processing Lambda
Validates QR code, checks payment status, retrieves artifact data from DynamoDB
↓
4. Content Check
Checks ContentCache table for pre-generated content
↓
5A. Cache Hit
Returns S3 URLs for audio, video, infographics → CloudFront CDN → User
OR
5B. Cache Miss
Triggers Content Generation Lambda → Bedrock AI → Polly TTS → S3 Storage → User

Admin Management Flow

1. Admin Login
Admin dashboard → Cognito authentication → JWT token
↓
2. API Request
Admin action (create temple, update artifact, etc.) → API Gateway
↓
3. Custom Authorizer
Validates JWT token, checks permissions, enforces rate limiting (100 req/min)
↓
4. Admin Backend Lambda
Processes request, validates data, performs operation on DynamoDB
↓
5. Audit Logging
Records action to AuditLog table (who, what, when, before/after values)
↓
6. Response
Returns success/error to admin dashboard with updated data

Temple Management (8 endpoints)

GET /admin/temples List with pagination, search, filters
GET /admin/temples/{siteId} Get single temple
POST /admin/temples Create temple
PUT /admin/temples/{siteId} Update temple
DELETE /admin/temples/{siteId} Soft delete temple
POST /admin/temples/{siteId}/images Upload images to S3
POST /admin/temples/bulk-delete Bulk delete temples
POST /admin/temples/bulk-update Bulk update temples

Artifact Management (8 endpoints)

GET /admin/artifacts List with pagination, search, filters
GET /admin/artifacts/{artifactId} Get single artifact
POST /admin/artifacts Create artifact + QR code
PUT /admin/artifacts/{artifactId} Update artifact
DELETE /admin/artifacts/{artifactId} Soft delete artifact
POST /admin/artifacts/{artifactId}/media Upload media to S3
GET /admin/artifacts/{artifactId}/qr-code Download QR (PNG/SVG/PDF)
POST /admin/artifacts/bulk-delete Bulk delete artifacts

Content Job Monitoring (5 endpoints)

GET /admin/content-jobs List jobs with filters
GET /admin/content-jobs/{jobId} Get job details + logs
POST /admin/content-jobs/{jobId}/retry Retry failed job
POST /admin/content-jobs/{jobId}/cancel Cancel in-progress job
GET /admin/content-jobs/stats Job statistics

Analytics (7 endpoints + export)

GET /admin/analytics/summary Key metrics dashboard
GET /admin/analytics/qr-scans QR scan statistics
GET /admin/analytics/content-generation Content gen stats
GET /admin/analytics/language-usage Language distribution
GET /admin/analytics/geographic Geographic distribution
GET /admin/analytics/audio-playback Audio playback stats
GET /admin/analytics/qa-interactions Q&A interaction stats
POST /admin/analytics/export Export CSV/JSON to S3
